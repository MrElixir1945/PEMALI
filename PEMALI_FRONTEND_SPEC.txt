1. PROJECT OVERVIEW
================================================================================

PEMALI adalah sistem multi-agent audit lingkungan Bali.
User kirim prompt вҶ’ Manager Agent rencana вҶ’ Sub-Agent eksekusi tool вҶ’
data dikumpulin вҶ’ Manager sintesis laporan final.

Stack:
  Backend  вҶ’ FastAPI (Python), OpenRouter LLM, PostgreSQL, ChromaDB
  Frontend вҶ’ Next.js 15 (App Router), TypeScript, Zustand, Framer Motion
  SSE      вҶ’ Real-time streaming backend вҶ’ frontend

File utama:
  backend/core/orchestrator.py    вҖ” Manager + SubAgent + 4-fase pipeline
  backend/core/models.py          вҖ” Pydantic models (TelemetryEvent, dll)
  backend/core/telemetry.py       вҖ” SSE emitter
  backend/main.py                 вҖ” FastAPI routes: POST /api/stream, GET /api/telemetry
  frontend/src/app/dashboard/page.tsx  вҖ” 2203 baris, SEMUA komponen di sini
  frontend/src/app/dashboard/page.backup.tsx вҖ” backup sebelum 4-fase refactor

2. HOW BACKEND SENDS DATA TO FRONTEND
================================================================================

2.1 Endpoint: POST /api/stream
    Body: { "prompt": "Audit lingkungan Gianyar..." }
    Response: text/event-stream (SSE)

2.2 SSE Format
    Dua tipe event, dibedakan oleh field "event:":

    a) event: state   вҶ’ TelemetryEvent (perubahan state agent/node)
       Format:  event: state\n
                data: {json}\n
                \n

    b) event: token   вҶ’ TokenEvent (LLM streaming token)
       Format:  event: token\n
                data: {json}\n
                \n

2.3 State Event Structure (TelemetryEvent)
    {
      "trace_id": "tr-abcd1234",        // ID sesi
      "node_id": "manager",             // ID node: "manager" | "geo_agent" | "water_agent" | tool_name
      "node_type": "Manager",           // "Manager" | "SubAgent" | "Module" | "System"
      "state": "THINKING",              // "IDLE" | "THINKING" | "SPAWNING" | "EXECUTING" | "DONE" | "ERROR"
      "narrative": "Menganalisis...",   // Teks narasi dari LLM / orchestrator
      "timestamp": 1715702400,          // Unix epoch seconds
      "metadata": {                     // Opsional, tergantung konteks
        "phase": "planning",            // Fase Manager: "planning"|"execute"|"validate"|"synthesis"|"done"
        "phase_step": "spawning",       // Sub-step: "spawning"|"progress"|"complete"|"checking"|"anomaly"|"re-spawn"|"re-spawn-done"|"valid"
        "task_count": 4,                // Jumlah agent yang di-spawn (execute spawning)
        "success_count": 3,             // Agent sukses (execute progress)
        "fail_count": 1,
        "done": ["geo_agent"],          // Agent yang sudah DONE (execute progress)
        "pending": ["water_agent"],     // Agent yang masih running (execute progress)
        "agent": "water_agent",         // Agent anomali (validate)
        "reason": "empty_data",         // Alasan anomaly (validate)
        "attempt": 1,                   // Re-spawn attempt (validate)
        "status": "success",            // "success"|"failed" (validate re-spawn)
        "type": "final_report",         // Tipe event (done)
        "session_id": "sess-..."        // Session ID (done, execute)
      }
    }

2.4 Token Event Structure
    {
      "node_id": "manager",             // Agent mana yang ngomong
      "content": "Berdasarkan data..."  // Chunk token dari LLM
    }

2.5 Complete Manager Event Sequence (per audit session)

FASE 1 вҖ” PLANNING (phase: "planning")
  event: state | node_id:"manager" | state:THINKING | phase:"planning"
  вҶ’ "Menganalisis permintaan user untuk audit Gianyar..."
  вҶ’ Node: LLM call ke OpenRouter dengan tool create_audit_plan
  вҶ’ Output: MasterPlan (daftar task + sub-agent yang harus dijalankan)

FASE 2 вҖ” EXECUTE (phase: "execute")
  event: state | node_id:"manager" | state:THINKING | phase:"execute" | phase_step:"spawning" | task_count:N
  вҶ’ "Menjalankan N agent: geo_agent, fire_agent, ..."

  // Sub-agents run (parallel atau sekuensial via DAG)
  // Per sub-agent:
  event: state | node_id:"geo_agent" | node_type:"SubAgent" | state:THINKING
  вҶ’ narasi dari LLM sub-agent
  event: state | node_id:"geo_agent" | state:EXECUTING
  вҶ’ "Menjalankan tool: mock_data_generator..."
  event: state | node_id:"mock_data_generator" | node_type:"Module" | state:EXECUTING
  event: state | node_id:"mock_data_generator" | state:DONE | metadata:{duration_ms, tool_name}
  event: state | node_id:"geo_agent" | state:DONE

  // Setiap sub-agent DONE вҶ’ Manager progress:
  event: state | node_id:"manager" | state:THINKING | phase:"execute" | phase_step:"progress" | done:[...] | pending:[...]
  вҶ’ "geo_agent selesai. Menunggu water_agent, fire_agent..."

FASE 3 вҖ” VALIDATE (phase: "validate") вҖ” BARU
  event: state | node_id:"manager" | state:THINKING | phase:"validate" | phase_step:"checking"
  вҶ’ "Memvalidasi data dari 4 agent..."

  // Kalau ada anomali:
  event: state | node_id:"manager" | state:THINKING | phase:"validate" | phase_step:"anomaly" | agent:"water_agent" | reason:"empty_data"
  вҶ’ "water_agent data kosong. Re-spawn..."
  event: state | node_id:"manager" | state:THINKING | phase:"validate" | phase_step:"re-spawn-done" | agent:"water_agent" | status:"success"
  вҶ’ "Re-spawn water_agent sukses."

  // Final validate:
  event: state | node_id:"manager" | state:THINKING | phase:"validate" | phase_step:"valid"
  вҶ’ "Semua data valid. Lanjut synthesis."

FASE 4 вҖ” SYNTHESIS (phase: "synthesis")
  event: state | node_id:"manager" | state:THINKING | phase:"synthesis"
  вҶ’ narasi sintesis dari LLM
  event: token | node_id:"manager" | content:"Berdasarkan data..."
  event: token | node_id:"manager" | content:"dari 4 agent..."
  вҶ’ Token streaming ~30-100 token events selama synthesis

FASE 5 вҖ” DONE (phase: "done")
  event: state | node_id:"manager" | state:DONE | phase:"done" | type:"final_report"
  вҶ’ Full report text (markdown)

2.6 SubAgent Event Sequence (per agent)

  event: state | node_id:"water_agent" | node_type:"SubAgent" | state:THINKING
  вҶ’ LLM narrative: "Saya akan analisa kualitas air..."
  // Bisa multiple THINKING events sebelum EXECUTING
  event: state | node_id:"water_agent" | state:EXECUTING
  вҶ’ "Menjalankan tool: mock_data_generator..."
  event: state | node_id:"mock_data_generator" | node_type:"Module" | state:EXECUTING
  event: state | node_id:"mock_data_generator" | state:DONE
  event: state | node_id:"water_agent" | state:DONE

  Variasi:
  - SubAgent bisa langsung DONE tanpa EXECUTING (kalau LLM jawab text-only)
  - SubAgent bisa ERROR (timeout, LLM error, tool error)
  - SubAgent bisa THINKING berkali-kali (LLM response multi-chunk)

2.7 Node IDs yang muncul

  Manager:   "manager"
  SubAgents: "geo_agent", "water_agent", "fire_agent", "osint_agent", "scheduler_agent"
  Modules:   "mock_data_generator", "system_scheduler"
  System:    "worker_daemon" (untuk autonomous tasks, jarang)

4. DATA FLOW PATTERNS (Yang perlu diketahui untuk redesign)

4.1 Event Arrival Order

  Manager Planning вҶ’ Manager Execute Spawn вҶ’
  [SubAgent-1 THINKING] [SubAgent-2 THINKING] [SubAgent-N THINKING] ...
  [SubAgent-1 EXECUTING] [SubAgent-2 EXECUTING] ...
  [Module-1 EXECUTING вҶ’ DONE] ...
  [SubAgent-1 DONE] [SubAgent-2 DONE] ...
  Manager Execute Progress (per agent) вҶ’
  Manager Validate вҶ’ (optional re-spawn loop) вҶ’
  Manager Synthesis (token stream) вҶ’
  Manager DONE вҶ’ final report

  CATATAN: Sub-agents dijalankan paralel via asyncio.gather().
  Frontend akan terima event interleaved (agent A THINKING, agent B THINKING,
  agent A EXECUTING, agent B EXECUTING, agent A DONE, agent B DONE).

4.2 Manager vs SubAgent events вҖ” Cara BEDAIN

  Manager:  node_id === "manager"
  SubAgent: node_type === "SubAgent" && node_id !== "manager"
  Module:   node_type === "Module"

4.3 Token Streaming

  Hanya saat Fase 4 Synthesis:
    event: token | node_id:"manager" | content:"chunk..."
  
  Frontend accumulator: store.tokens["manager"] = semua chunk digabung.
  ChatMessages menampilkan tokens["manager"] sebagai streaming text.

4.4 Chat Mode vs Audit Mode

  Chat mode (greeting / simple question):
    Manager THINKING вҶ’ Manager DONE (metadata.type TIDAK "final_report")
    вҶ’ Tidak ada SPAWNING, tidak ada SubAgent

  Audit mode (ada MasterPlan):
    Full 4-fase pipeline seperti di atas
    вҶ’ Manager DONE dengan metadata.type === "final_report"

4.5 Re-spawn Flow

  Saat Validate deteksi anomali:
    1. Manager emit THINKING phase:validate + phase_step:anomaly
       вҶ’ narrative: "water_agent error: data kosong. Re-spawn..."
    2. Backend re-spawn SubAgent water_agent
       вҶ’ SubAgent events: THINKING вҶ’ EXECUTING вҶ’ DONE (seperti biasa)
    3. Manager emit THINKING phase:validate + phase_step:re-spawn-done
       вҶ’ narrative: "Re-spawn water_agent sukses."

8. SSE PARSING вҖ” Kode Persis (JANGAN diubah)

    const reader = res.body.getReader();
    const decoder = new TextDecoder();
    let buffer = "";
    while (true) {
      const { done, value } = await reader.read();
      if (done) break;
      buffer += decoder.decode(value, { stream: true });
      const lines = buffer.split("\n");
      buffer = lines.pop() ?? "";
      let eventType = "";
      let dataLine = "";
      for (const line of lines) {
        if (line.startsWith("event:")) {
          eventType = line.slice(6).trim();
        } else if (line.startsWith("data:")) {
          dataLine = line.slice(5).trim();
        } else if (line === "" && dataLine) {
          const parsed = JSON.parse(dataLine);
          if (eventType === "state") addEvent(parsed);
          else if (eventType === "token") addToken(parsed.node_id, parsed.content);
          eventType = ""; dataLine = "";
        }
      }
    }
